**Какие jQuery-функции использовались?**

В работе использовались следующие ключевые функции и методы jQuery (включая jQuery UI):

- $(document).ready() — ожидание готовности DOM.
- $() — селектор элементов (например, $('.tab-btn'), $('#loadQuote')).
- .click() и .on('click') — обработка кликов.
- .removeClass(), .addClass() — работа с классами для переключения табов.
- .data('tab') — получение значения data-атрибута.
- .attr('href') — получение атрибута href.
- .animate({ scrollTop: ... }) — плавная анимация прокрутки.
- $.get(url) с .done() и .fail() — AJAX-запрос для загрузки цитаты.
- .draggable() (из jQuery UI) — перетаскивание элемента.
- .datepicker({ ... }) (из jQuery UI) — виджет календаря с параметрами (dateFormat, changeMonth, changeYear и др.).

Эти функции упростили манипуляции с DOM, события и AJAX по сравнению с чистым JavaScript.

**Что показалось быстрее/проще — jQuery или чистый JS?**

JQuery оказался проще и быстрее в написании кода:

1) Одна строка вроде $('#draggable').draggable(); даёт полноценный drag-and-drop.
2) Цепочки методов и неявная итерация (например, .removeClass('active') сразу для всех элементов) экономят десятки строк.
3) AJAX через $.get() короче и удобнее, чем цепочка fetch().then().then().

Чистый JS (Vanilla) требует больше кода и явных циклов (forEach, classList, dataset), но он лаконичнее в современных задачах (например, прокрутка через scrollIntoView({ behavior: 'smooth' }) — одна строка против .animate()).
В целом, для простых задач (табы, события) jQuery проще для новичков, но Vanilla JS быстрее выполняется и не требует библиотеки.

**Что из этого вы бы оставили, если писали современный сайт?**
Если бы я писала современный сайт:

- Я оставила бы чистый JavaScript (Vanilla JS) для большинства задач: табы, события, работа с классами — через querySelectorAll, addEventListener, classList.
- Плавную прокрутку — через нативный scrollIntoView({ behavior: 'smooth' }).
- AJAX — через fetch (или async/await).
  
Это легче, быстрее загружается, соответствует современным стандартам и не требует внешних библиотек.

- jQuery использовала бы только в legacy-проектах (поддержка старого кода, WordPress-темы, плагины).
- jQuery UI — заменила бы на современные альтернативы (например, библиотеки вроде interact.js для drag-and-drop или flatpickr для календаря), либо на фреймворки (React/Vue с готовыми компонентами).
